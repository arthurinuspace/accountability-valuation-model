# 當責評分參考手冊 (Evaluation Guidebook)

## 📘 手冊指引

這份手冊是為了**確保評分的公正性與一致性**而設計。這不是一份隨意打分數的表格，而是一份**「請你證明你的價值」**的檢核清單。

**適用對象**：
*   **評分者 (Scorers)**：如 PM（評 IT）、主管（評部屬）。請據此標準給出客觀分數。
*   **被評者 (Employee)**：請據此標準蒐集自己的工作成果作為佐證。

---

## 🔢 評分公式速查

$$
Total = (\underbrace{Clarify\\% \times Execute\\%}_{Base}) \times 200 + \underbrace{(SOP\\% \times 100) + (Beyond\\% \times 100)}_{ValueAdd} + Risk(0-20)
$$

> 註：GitHub 對複雜 LaTeX 支援度有限，若上方公式無法完整呈現，請參考下方純文字版：
>
> **總分 = (釐清% × 執行%) × 200 + (作法改善% × 100 + 超越目標% × 100) + 風險補償**

---

## 🎯 維度一：目標釐清 (Clarify)【乘法關鍵】

**定義**：在動手之前，是否主動確認了「為什麼要做」、「成功標準是什麼」。
*權重邏輯：沒有釐清，直接歸零。*

| 分數 (Multi) | 定義 | 具體行為範例 (Behavioral Anchors) |
| :--- | :--- | :--- |
| **0** | **未對齊 / 盲目執行** | ❌ 老闆隨口說一句就直接做，結果做出來完全不是要的東西。<br>❌ 發現規格有明顯邏輯錯誤，卻「照單全收」不提出疑問。 |
| **0.5 ~ 0.8** | **被動接受** | ⚠️ 有參加 Kick-off 會議，但全程沈默，只會在會後私下抱怨做不到。<br>⚠️ PM 給了規格書，但自己完全沒看，邊做邊問重複的問題。 |
| **1.0** (合格) | **主動確認** | ✅ <ins>**開工前**</ins> 發出了確認信（Email/Jira），重述了一次目標並獲得認可。<br>✅ 清楚知道 Deadline 與驗收標準 (Acceptance Criteria)。 |
| **1.1 ~ 1.2** | **反向糾錯 / 優化** | 🌟 **在需求階段就發現問題**：「這個流程會導致資料庫鎖死，建議改成...」<br>🌟 協助主管/PM 把模糊的需求轉化為具體的技術規格。 |

---

## 🔨 維度二：任務執行 (Execute)

**定義**：在既定的時間與規格內，交付成果的品質。
*權重邏輯：這是基本盤。*

| 分數 (Multi) | 定義 | 具體行為範例 (Behavioral Anchors) |
| :--- | :--- | :--- |
| **0 ~ 0.5** | **嚴重失職** | ❌ 無法交付，且無提前預警。<br>❌ 產出品質低劣（Bug 連發），完全無法上線。 |
| **0.8** | **勉强達標** | ⚠️ 雖然做完了，但遲交。<br>⚠️ 功能會動，但效能很差或代碼髒亂 (Spaghetti Code)，即便符合功能需求。 |
| **1.0** (合格) | **精準交付** | ✅ **On Time, On Spec, On Quality**。<br>✅ 交付的東西與當初「釐清」的一模一樣，測試一次通過。 |
| **1.1 ~ 1.2** | **超水準交付** | 🌟 提早交付。<br>🌟 在滿足功能的同時，還主動兼顧了極致的效能或擴充性。 |

---

## 📚 維度三：作法改善 (SOP Improvement)

**定義**：將隱性經驗顯性化，建立「可被傳承、可被複用」的資產。
*計分邏輯：這是加法（增值）。沒有做就是 +0，有做就是賺到。*

| 分數 (Add) | 等級 | 具體行為範例 (Engineering / General) |
| :--- | :--- | :--- |
| **+0%** | **無資產產出 (維持現狀)** | ⚪️ 事情做完了，但沒留下任何文件。只有你自己知道怎麼維護。 |
| **+1% ~ +39%** | **初步嘗試 (Seed)** | 🌱 主動整理個人筆記或工作文件。<br>🌱 提出改善建議，展現主動性。 |
| **+40% ~ +99%** | **具體優化 (Sprout)** | 🌿 建立部門內可用的 SOP，減少重複失誤。<br>🌿 實現小規模創新，優化既有流程。 |
| **+100% ~ +149%** | **組織資產 (Tree)** | 🌳 建立跨部門通用系統，顯著提升全公司效率。<br>🌳 帶來額外營收或大幅降低成本。 |
| **+150% ~ +199%** | **產業標竿 (Forest)** | 🦄 建立業界領先的方法論，成為公司護城河。<br>🦄 取得專利、獲獎或開創全新商業模式。 |
| **+200%** | **重新定義 (Cosmos)** | 🪐 重新發明了行業標準，定義了未來十年的運作模式。 |

---

## 🚀 維度四：超越目標 (Beyond Goal)

**定義**：在原定範疇之外，主動創造了額外的價值或洞察。
*計分邏輯：這是驚喜（Bonus）。*

| 分數 (Add) | 等級 | 具體行為範例 |
| :--- | :--- | :--- |
| **+0%** | **本分完成 (維持現狀)** | ⚪️ 完成了交辦事項，不多也不少。 |
| **+1% ~ +39%** | **點子萌芽 (Seed)** | 🌱 (UI) 主動加了微動畫，讓使用者體驗更好。<br>🌱 (Backend) 發現某些 Query 稍慢，主動提出優化建議。 |
| **+40% ~ +99%** | **具體產出 (Sprout)** | 🌿 透過快取優化，讓 Response Time 減少 50%。<br>🌿 分析 Log 發現潛在使用者的行為模式，向 PM 提出新功能建議。 |
| **+100% ~ +149%** | **顯著價值 (Tree)** | 🌳 帶來的優化直接節省了 20% 的雲端成本。<br>🌳 建議的新功能上線後，成為產品的核心賣點之一。 |
| **+150% ~ +199%** | **卓越貢獻 (Forest)** | 🦄 用新技術解決了公司長年無法解決的效能瓶頸。<br>🦄 該專案獲得外部大獎或專利。 |
| **+200%** | **重新定義 (Cosmos)** | 🪐 創造了全新的市場或技術典範，改變了人類的生活方式。 |

---

## 🛡️ 維度五：風險補償 (Risk Compensation)

**定義**：在面對不確定性或危機時的擔當與應變。
*計分邏輯：這是保險（Insurance）。*

| 分數 (Add) | 等級 | 具體行為範例 |
| :--- | :--- | :--- |
| **0** | **無風險/逃避** | 😶 專案順風順水，無風險發生。<br>❌ 遇到問題隱瞞不報，導致最後炸鍋。 |
| **+5** | **主動預警** | 📢 「我發現這個依賴庫下個月會停止維護，我們需要提早規劃...」<br>📢 誠實回報進度落後，並提出 Plan B。 |
| **+10** | **滅火 (Firefighting)** | 🔥 發生 P0 Incident，半夜起來修復上線，把損失降到最低。 |
| **+20** | **系統性防災** | 🛡️ 經歷一次災難後，寫了 Post-mortem (檢討報告)，並實作了防呆機制，**確保也就是說這種錯永遠不會再發生**。 |

---

## 📢 給評分者的建議 (Tips for Scorers)

1.  **舉證責任 (Burden of Proof)**：不要憑感覺打分。請員工提供連結（Jira, Wiki, Git Commit, Email）。
    *   *「你說你有釐清目標，請給我看那封確認信。」*
    *   *「你說你有改善 SOP，請貼連結給我。」*
2.  **區分「苦勞」與「資產」**：
    *   如果員工加班到很晚，但沒有產出 SOP，這只能算在「執行」分（如果品質好），不能算在加法區。
3.  **大膽給零分**：
    *   如果目標真的沒釐清，請勇敢在「釐清」項打 0 或 0.5。這會讓總分為 0。這是一個強烈的訊號，能有效矯正組織行為。

## 🏆 總分與人才分級對照 (Total Score & Levels)

| 總分區間 | 等級稱號 | 定義 |
| :--- | :--- | :--- |
| **0 - 199** | **⚠️ 需改善 (Underperformer)** | 連基本盤都未能守住。可能存在釐清不足 (做白工) 或執行力嚴重落差。 |
| **200 - 279** | **✅ 稱職 (Competent Performer)** | 守住了基本盤 (200分)，但在增值盤未有貢獻。是一個可靠的執行者。 |
| **280 - 399** | **⭐ 實踐者 (Practitioner)** | 中堅力量。除了守住基本盤，也開始在「增值盤」有所貢獻 (SOP/創新)，值得培養。 |
| **400 - 549** | **🚀 槓桿者 (Lever)** | 一人抵多人用。高基本盤 + 高增值盤，為組織帶來了槓桿效應。 |
| **550+** | **👑 傳奇 (Legend)** | 重新定義了遊戲規則！其貢獻已超越職位框架，創造了巨大的商業價值與系統資產。 |
